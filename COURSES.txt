# COURSE DATA FILE
# =================
# This file is the single source of truth for all course information
# used to build the individual course-*.html pages.
#
# FORMAT OVERVIEW
# ----------------
#
# Each course is a block starting with `COURSE:` and separated by a line with `---`.
#
# Fields per course (all text is plain, no HTML needed):
#   COURSE       A short ID that also appears in the HTML filename, e.g. "course-real-analysis".
#   Title        Full course title as shown on the course page.
#   ShortName    Short name used on the Teaching overview page.
#   Level        Target audience, e.g. "Integrated MSc Students".
#   Institution  e.g. "University of Hyderabad".
#   Semesters    One or more semesters, separated by commas.
#   Books        One book per line, each starting with "- ".
#   Syllabus     Free-text block delimited by <<< and >>>.
#   ProblemSets  One problem set per line, each starting with "- ", giving label and suggested PDF file name.
#
# Example ProblemSets line:
#   - Practice Set 1 | course-real-analysis-ps1.pdf
#
# Suggested PDF naming pattern (you may adapt as needed):
#   <COURSE>-[ps/notes/handout]<number>.pdf
# Example: course-real-analysis-ps1.pdf, course-math-ii-notes1.pdf
#
# When you add a new course here, you should:
#   1) Copy an existing course-*.html file.
#   2) Rename it and update its contents using the values from this file.
#   3) Make the course name on teaching.html link to that new HTML file.
#
